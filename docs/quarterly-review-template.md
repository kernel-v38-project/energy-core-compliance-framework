# Quarterly Review Template

## Purpose

This template is used to run quarterly reviews of the energy_core Enterprise trust center, compliance documentation, and public release posture.

## Review scope

Review the following areas:
- README.md;
- index.html;
- RELEASE_NOTES.md;
- docs/compliance.md;
- docs/privacy.md;
- docs/dpa.md;
- docs/subprocessors.md;
- docs/incident-response.md;
- docs/architecture.md;
- docs/security-questions.md;
- docs/change-log-policy.md;
- docs/public-release-process.md;
- docs/roles-and-owners.md;
- docs/review-cadence.md;
- docs/audit-evidence.md;
- docs/evidence-index-template.md;
- .github/release.yml.

## Review goals

- Confirm that public content is current.
- Verify that evidence still supports public claims.
- Identify stale, conflicting, or incomplete statements.
- Track changes that need follow-up.
- Keep ownership and release processes aligned.

## Review checklist

### Public trust center
- README.md is in English and matches index.html.
- Links resolve correctly.
- Public wording is current and consistent.
- No placeholder or outdated text remains.
- Buyer-facing claims are supported by evidence.

### Security
- SECURITY.md reflects the current security posture.
- Incident response language is current.
- Any security-related public claims are still valid.
- Security evidence is current and accessible.

### Privacy and compliance
- docs/privacy.md reflects current data handling.
- docs/dpa.md matches the current contract posture.
- docs/compliance.md reflects the current compliance position.
- Any legal or regulatory wording changes have been approved.

### Subprocessors
- docs/subprocessors.md is current.
- All listed vendors are still active and correct.
- Region, purpose, and transfer notes are accurate.
- Any changes are reflected in release notes.

### Architecture
- docs/architecture.md matches the current platform structure.
- Deployment notes are current.
- Any material architectural changes are documented.

### Release and process
- RELEASE_NOTES.md is current.
- .github/release.yml still matches the label taxonomy.
- docs/change-log-policy.md is current.
- docs/public-release-process.md is current.
- docs/roles-and-owners.md is current.
- docs/review-cadence.md is current.

### Evidence
- Evidence items still map to current claims.
- New evidence has been logged.
- Superseded evidence has been marked appropriately.
- Evidence access levels are still correct.
- File paths and references still work.
